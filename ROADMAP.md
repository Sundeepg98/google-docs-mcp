## Top 5 highest-leverage

These unlock the most value (or the most risk reduction) per unit of effort, across all categories:

1. ✅ **DONE (#153) — Generalize the Sheets `batchUpdate` request-builder** *(Architecture, P1/L)* — one small typed helper unlocks ~8 Sheets feature gaps at once (formatting, conditional formatting, charts, pivots, sheet-lifecycle, dimensions). The deferral rationale ("no precedent") was stale: `batchUpdate` already ships in docs + slides. **Shipped by #153**: `services/sheets/batch.py` — pure typed builders (`grid_range`, `color`, `cell_format`, `repeat_cell_request`, `add_conditional_format_rule_request`) + an `execute_with_retry`-wrapped `batch_update` dispatcher, mirroring the docs/slides pattern. Proven end-to-end by ONE wired op (`gsheets_format_range` → `repeatCell`); golden re-frozen 39→40. Remaining ~7 features (conditional-format/chart/dimension/sheet-lifecycle TOOLS) now layer on the seam — see Features-P1/P2 Sheets items.
2. ✅ **DONE (#14 / #74) — Derive the OAuth consent union from per-service scope declarations** *(Hardening, P1/M)* — was TWO hand-synced central scope lists (`auth.SCOPES` + `oauth_google.GOOGLE_API_SCOPES`), a classic drift bug (stdio works while HTTP 403s). **Now single-source:** `auth.WORKSPACE_SCOPES` is the one source of truth; `auth.SCOPES = WORKSPACE_SCOPES` (stdio) and `oauth_google.GOOGLE_API_SCOPES = [*IDENTITY_SCOPES, *WORKSPACE_SCOPES]` (HTTP/connector) both derive from it, so adding a service scope is a one-line edit. Guarded by `tests/unit/test_scope_union_single_source.py` (frozenset-equality so drift fails CI) + `test_base_tier_scopes.py` (no restricted-scope creep). The single genuine multi-service seam — closed.
3. ✅ **DONE (#152) — socket/transport timeout on the Google API client** *(Hardening, P1/S)* — `build()` used default httplib2 with NO deadline; a stalled connection hung `.execute()` forever, defeating the retry layer. **Fixed by #152**: the chokepoint adapter now builds a deadline-bearing `AuthorizedHttp` (default 30s, env `GOOGLE_API_HTTP_TIMEOUT_SECONDS`) and the retry predicate treats transport timeouts as retryable so backoff actually engages on a hang. See Hardening-P1 for detail.
4. **Move PKCE verifier + NonceStore into the shared StorageBackend** *(Architecture, P1/M)* — the one true affinity bug on the live OAuth path. Harmless on single-machine Fly, but becomes systemic intermittent-login failure the moment a 2nd instance or Vercel cold-start is added — exactly the documented scale-out plan.
5. ✅ **DONE (#150) — Wire the frame-upload POST handler into the route table** *(Features, P1/S)* — `upload_frame_endpoint` was defined but never routed, so `POST /upload/frames/*` 404'd (a real prod regression introduced by #148). **Fixed by #150**: route wired before the `Mount('/')` catch-all + a route-presence integration test added; merged + deployed; live POST probe returns **403 not 404** (route registered). *Promoted into this slot as the next-highest open action: **Generalize the Apps Script automation library into a `recipes/` registry** — see Features-P3 / Architecture-P3 (`as_list_recipes`/`as_install_recipe`), the highest-leverage remaining feature lever now that the route regression is closed.*

## Hardening

### P1
- ✅ **DONE (#152) — socket/transport timeout on the Google API client** — `build()` used default httplib2 (no deadline); a hang defeated retry, *[P1/S/high]* — **Fixed by #152**: `GoogleApiClientAdapter.get_service` now builds the SDK client with `build(..., http=AuthorizedHttp(credentials, http=httplib2.Http(timeout=N)))` — a connect+read deadline (default **30s**, env-overridable via **`GOOGLE_API_HTTP_TIMEOUT_SECONDS`**, finite-positive-validated). The retry predicate was extended (`_is_retryable_transport_error`) so a `socket.timeout`/`TimeoutError`/`ConnectionError`/`ETIMEDOUT`-`ECONNRESET`-`ECONNABORTED` now triggers the existing exponential backoff on idempotent calls (and still never replays a non-idempotent mutation). Auth path untouched — `AuthorizedHttp` wraps the already-resolved credentials unchanged; only the transport socket gets a deadline. Tests: `tests/unit/test_google_api_timeout.py`.
- ✅ **DONE (#14 / #74) — OAuth consent now generalizes from one source** — was two hand-synced central scope lists (`auth.SCOPES` + `oauth_google.GOOGLE_API_SCOPES`) edited BY HAND for every new service. **Resolved by single-sourcing on `auth.WORKSPACE_SCOPES`:** `auth.SCOPES = WORKSPACE_SCOPES` (stdio) and `oauth_google.GOOGLE_API_SCOPES = [*IDENTITY_SCOPES, *WORKSPACE_SCOPES]` (HTTP/connector) both derive from it (`oauth_google` imports `WORKSPACE_SCOPES` from the leaf `auth` module, no cycle), so adding a service scope is a one-line edit and the stdio/HTTP sets can't drift. Pinned by `tests/unit/test_scope_union_single_source.py` (frozenset equality) and `test_base_tier_scopes.py` (the in-repo no-CASA / no-restricted-creep guard). **The one genuine multi-service seam — closed.**

### P2
- **Several `.execute()` call sites are NOT retry-wrapped** — legacy `AppsScriptClient` (gas_deploy, 5/0), `retrofit.py` (3/0), `docx_import.py` (1/0), docs multi-step create (10/19), *[P2/M/high]* — decompose `AppsScriptClient` into idempotent sub-steps and wrap the read-shaped ones with `execute_with_retry(idempotent=True)`; wrap idempotent reads in retrofit/docx_import; explicitly mark `make_doc_with_tabs`/`add_tabs_to_doc` as intentionally single-shot (blanket retry across a partial mutation is unsafe).
- **No corruption detection; unguarded DB open at import crashes boot on a corrupt file** — `new Database(dbFile)` at module load, no `PRAGMA integrity_check`, *[P2/M/high]* — wrap the open in try/catch, run `PRAGMA quick_check` on startup, auto-restore the newest `backups/` file on failure before re-opening. *(SQLite dev/test layer.)*
- **Unconditional `VACUUM` on every startup** — a full-DB rewrite runs at every boot regardless of pending migrations (two sites, incl. `startServer`), *[P2/S/high]* — gate VACUUM on a migration actually applying and/or move it to the maintenance job; remove the unconditional `startServer` VACUUM.
- **SQLite migration failure logs and CONTINUES** — can boot on a half-applied schema (Postgres correctly fails fast), *[P2/S/high]* — rethrow inside the SQLite migration catch so a failed migration aborts boot.
- **Backup/snapshot/export use live-file `copyFileSync` instead of `.backup()`** — risks stale/torn copies (uncommitted `-wal` frames excluded), *[P2/M/high]* — use better-sqlite3 `db.backup(dest)` for snapshot + backup script; for export, `db.backup()` to temp or `PRAGMA wal_checkpoint(TRUNCATE)` before download.
- **`restore-db.js` auto-overwrites with the newest backup with NO confirmation** — accidental `npm run restore` silently clobbers the live DB, *[P2/S/high]* — add a readline y/N gate (bypassable via `--yes`) before the overwrite; make the pre-restore safety copy via `db.backup()`.
- ✅ **DONE** - **Frame-upload staging size/count cap at the write boundary** - `stage_frame_bytes` enforces a per-frame byte cap (`_MAX_FRAME_BYTES` = 10 MiB), a per-batch count cap (`_MAX_FRAMES_PER_BATCH` = 200) and a cumulative-byte cap (`_MAX_BATCH_TOTAL_BYTES` = 250 MiB) before write, independent of Content-Length (`services/apps_script/_frames_staging.py:98-100, 249-272`); the route also hard-caps the streamed body (`http_server/routes/convert.py:141-154`).
- **`TrustedHost` fails OPEN (`['*']`) when no TRUSTED_HOSTS/FLY_* env is set** — affects the Vercel deploy target, *[P2/S/high]* — extend the existing FLY_REGION fail-closed guard to also detect Vercel (`VERCEL=1`) and refuse `['*']`, or require `TRUSTED_HOSTS` whenever `GOOGLE_OAUTH_BASE_URL` is set. *(Bounded: OAuth base URL is normally pinned.)*
- **Single-OAuth-client coupling across the two grants is undefended** — no startup assertion the two `client_id`s match (divergent resolver precedence + a one-sided stdio fallback), *[P2/S/high]* — at HTTP boot, resolve `client_id` via BOTH paths and assert equality, raising on mismatch (mirrors the existing persistence guard). A silent divergence breaks the sub-join with no clear error.
- **Metrics: only in-process `/info` counters; no Prometheus/OTel/statsd or scrapeable `/metrics`** — *[P2/M/medium]* — when scaling horizontally, mount `prometheus_client` (or OTel metrics) exposing a request-latency histogram + error-rate counter at a bearer-gated `/metrics`. Defensible to defer at single-machine scale (flag as known gap).
- **Alerting: no in-repo alert path (delegated to Sentry + Fly dashboards)** — delivery is unverifiable from code, *[P2/S/medium]* — document the required Sentry issue-alert rule + Fly health-check policy in the runbook as an explicit operator-config dependency. *(Sentry capture is init-only; events flow via the ERROR-level LoggingIntegration.)*
- **Health-check interval is 5m on the single node** — a half-broken node can serve errors ~5min with no peer, *[P2/S/high]* — under real load, drop the Fly interval to 30–60s paired with `min_machines_running>=2` (needs the KV backend + PKCE fix); reconcile the Dockerfile 30s vs Fly 5m divergence to one source of truth. *(Deliberate cost trade-off today.)*
